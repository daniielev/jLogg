jLogg

A simple Facebook Login Tool

A simple jQuery Plugin that allow you to login into your web app using the Facebook Javascript SDK.

Designed for web developers

Saves time

Works on Every Environment

It doesn't matter if you're building an app in Python, Ruby on Rails, AngularJS, or any other framework and backend. jLogg is very simple and will run along with your own app.

Login

Link JS

Move the dist/jlogg.min.js file to your server and link to it before the closing body tag.

Please note that you must include jQuery before the jLogg JS file.

<script src="//code.jquery.com/jquery-1.11.2.min.js"></script>
<script src="dist/jLogg.min.js"></script>

The Markup

You need to define 3 elements:

  • The main container, where your custom buttons live.
  • The custom facebook login and logout buttons, using HTML5 data-action attributes. I'll show you how...
<form id="login">
....
<button class="jlogg-custom-btn" data-action="facebook-login">Login using Facebook</button>
<button class="jlogg-custom-btn" data-action="facebook-logout">Logout</button>
....
</form>

Call Plugin

Just before the closing body tag (after any JS file includes), call the plugin with code below:

<script>
$(document).ready(function () {
    $("form#login").jLogg();
});
</script>

Note that the js code below will only initilize the plugin. It will not run until you send it some options. See documentation.

Plugin Settings

Plugin accepts settings from an object of key/value pairs, and can be assigned to any HTML element.

Property Default Description
appId String: "" You must enter you Facebook Application ID here. When you register your app, Facebook will generate an unique code for your app. You can set up your Facebook app from here https://developers.facebook.com/apps.
defaultButton Boolean: false

Optional

As false, then jLogg will go through your container element and look for 2 elements:

  1. Your custom facebook login button, which is set as the elemen using data-action="facebook-login".
  2. And your own facebook logout button, using data-action="facebook-logout".

But if you set this option as true, then you don't need to create custom login and logout functions. jLogg will load the Facebook Login Button and set it up for you.
buttons Object: {}

Optional

This property contains some other properties in an object object, which are used to customize the Facebook Login Button when you set the defaultButton to true.

Right now, it only takes one property: the size of the button. Some possible values are small, medium, large, xlarge. Here's the default object:

button : {
            size : "medium"
}

The features coming up as based on the Facebook Login Button reference, see https://developers.facebook.com/docs/plugins/login-button.
permissions String: "public_profile, email" Facebook uses these permissions (comma separated) for the scope of the FB object whenever you required the API, which is fill when you get the response from the Facebook SDK. Check what you can required from a logged in user on https://developers.facebook.com/docs/facebook-login/permissions/v2.3#reference.
onLogin Type: function

When the user has enter his Facebook credentials and completed the login process, onLogin function will be called. Here you have access now to the api, which is a JSON with all the properties you requested from the user using the permissions option.

Eg:

onLogin : function (api) {
  // console.log(api)
  alert("Welcome, " + api.name + "!");
}
onLogout Type: function

As the onLogout function, this is a callback. In this case, onLogout function will be executed when an already logged in user request to the FB object the logout method. Once the response.status is equal to unknow, the user has been successfuly logged out from your app.

It takes no arguments because when the function is called, the connection has been already closed.

Eg:

onLogout : function () {
  // Redirect to your app login page
  window.location = "/login";
}
